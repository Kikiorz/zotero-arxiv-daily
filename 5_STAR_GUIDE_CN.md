# 5 星评分系统使用指南

## 快速开始

### 在 Zotero 中给论文打星级

使用 5 星评分系统标记你 Zotero 文献库中的论文：

- **⭐⭐⭐⭐⭐** (5 星) - 极其相关，核心论文（权重 5.0x）
- **⭐⭐⭐⭐** (4 星) - 非常相关（权重 4.0x）
- **⭐⭐⭐** (3 星) - 相关（权重 3.0x）
- **⭐⭐** (2 星) - 有些相关（权重 2.0x）
- **⭐** (1 星) - 略微相关（权重 1.5x）
- **无标签** - 默认权重（权重 1.0x）

### 如何在 Zotero 中添加星级标签

1. 在 Zotero 中选择一篇论文
2. 在右侧面板找到"标签"（Tags）部分
3. 点击"添加"，输入星级标签（如 `⭐⭐⭐⭐⭐`）
4. 按回车保存

**提示**：你也可以使用文字标签：`5-star`, `4-star`, `3-star`, `2-star`, `1-star`

## 推荐工作流程

### 第一步：初始化你的评分系统

给你 Zotero 库中最重要的论文打分：

1. **5 星** ⭐⭐⭐⭐⭐：你研究领域的核心论文（10-20 篇）
   - 例如：你论文的主要参考文献
   - 你想要跟进的研究方向

2. **4 星** ⭐⭐⭐⭐：非常相关的论文（20-30 篇）
   - 例如：相关方法、类似问题的解决方案

3. **3 星** ⭐⭐⭐：相关论文（30-50 篇）
   - 例如：背景知识、相关领域的工作

4. **2 星及以下**：可选，用于降低某些论文的影响

### 第二步：每日使用

1. **查看邮件**：每天收到推荐的论文列表
2. **阅读感兴趣的论文**
3. **添加到 Zotero**：把感兴趣的论文加入 Zotero
4. **打星级**：根据相关性给新论文打星级
5. **系统学习**：随着你的文献库增长，推荐会越来越准确

### 第三步：定期维护

- **每周**：回顾新添加的论文，确保星级准确
- **每月**：调整旧论文的星级（研究兴趣可能变化）
- **随时**：发现特别重要的论文立即打 5 星

## 配置示例

### 基础配置（使用 API，速度快）

```yaml
zotero:
  user_id: ${oc.env:ZOTERO_ID}
  api_key: ${oc.env:ZOTERO_KEY}
  include_path: null  # 使用所有论文

executor:
  reranker: api  # 使用 API 模式（推荐）
  max_paper_num: 100  # 每天推荐 100 篇
  max_workers: 8  # 并发处理
  source: ['arxiv']  # 数据源

reranker:
  api:
    key: ${oc.env:OPENAI_API_KEY}
    base_url: ${oc.env:OPENAI_API_BASE}
    model: BAAI/bge-m3  # embedding 模型
    batch_size: 100
```

### 自定义星级权重

如果你想调整星级的权重：

```yaml
reranker:
  tag_weights:
    ⭐⭐⭐⭐⭐: 10.0  # 大幅提高 5 星论文的影响
    ⭐⭐⭐⭐: 6.0
    ⭐⭐⭐: 3.0
    ⭐⭐: 1.5
    ⭐: 1.0
```

## 推荐算法说明

最终推荐分数 = Σ(相似度 × 时间衰减 × 星级权重) / 总权重

**示例**：
- 新论文 A 与你的 5 星论文相似度高 → 高分推荐
- 新论文 B 与你的 2 星论文相似度高 → 低分推荐
- 最近添加的论文影响更大（时间衰减）

## 常见问题

**Q: 我需要给所有论文打星级吗？**
A: 不需要。只给最相关的论文打星级即可。未打星级的论文使用默认权重 1.0。

**Q: 5 星论文太多会怎样？**
A: 如果大部分论文都是 5 星，系统会推荐与这些论文相似的内容。建议只给真正核心的论文打 5 星。

**Q: 可以用其他标签吗？**
A: 可以！在配置中添加自定义标签权重：
```yaml
reranker:
  tag_weights:
    my-core-research: 5.0
    related-work: 3.0
```

**Q: 多久能看到效果？**
A: 当你标记 20-30 篇论文后，推荐就会明显改善。

**Q: GitHub Action 上会很慢吗？**
A: 使用 `api` reranker 模式，处理 100 篇论文通常在 5-10 分钟内完成。
